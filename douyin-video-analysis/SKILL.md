# Douyin Video Analysis

This skill exists to stop fake analysis and stop machine-gibberish from being shown to humans.

## Hard rules

1. Do not generate a per-video analysis markdown unless the local video file exists and can be probed.
2. Do not show the user low-value JSON full of `unknown` fields as if that were the analysis result.
3. Separate machine evidence from human report.
4. A report for humans must answer:
   - 这条视频讲了什么故事
   - 用了哪些手法
   - 这些手法起到了什么作用
   - 为什么数据会高
   - 镜头语言是什么
   - 分镜怎么拆
   - 剧本/台词能重构到什么程度
   - 后续如何复刻

## Required workflow

1. Verify local video exists.
2. Run `scripts/analyze_local_video.py`.
3. Read:
   - `analysis_md/<video_id>.md` as the human-readable report
   - `analysis_assets/<video_id>/analysis_evidence.json` as the machine evidence pack
   - `analysis_assets/<video_id>/contact_sheet.jpg`
   - `analysis_assets/<video_id>/keyframes/frame_*.jpg`
4. When talking to the user, prefer the long report. Only expose JSON evidence if the user explicitly asks.

## Output standard for humans

The final report should not feel like metadata commentary.

It should be long-form, evidence-backed, and useful enough that a human can:
- understand the video's narrative and emotional engine
- reuse its hook and structure
- derive a reproducible shooting recipe
- know what still needs manual or multimodal verification

If the report cannot support downstream recreation, it is not good enough.

## Review principle

The human report can contain hypotheses, but hypotheses must be clearly marked.
Never disguise missing transcript or missing frame-level semantics as certainty.
